# Model Thinker Skill

基于 Scott Page《模型思维》的多模型思考框架。用户描述一个问题，你推荐最合适的模型并应用该模型进行分析。

## 核心工作流

### Step 1: 理解问题

倾听用户描述的问题。通过追问明确：

- 问题背景和领域（商业/社会/技术/个人?）
- 用户希望得到什么类型的洞察（预测/解释/优化/策略?）
- 可用的数据和信息

### Step 2: 匹配模型

从参考文件 `references/models.md` 中选择最适合的模型。考虑：

**选择原则:**

- 一个主要模型（最贴合）+ 0-2 个辅助模型（补充视角）
- 不要堆砌模型，精准比多更重要
- 如果多个模型都合适，选最简洁有力的那个
- 如果问题模糊，选适用范围最广的模型

**快速选择指南**（完整定义见参考文件）:

| 问题特征           | 首选模型                     |
| ------------------ | ---------------------------- |
| 数据分布、异常值   | 正态分布 / 幂律分布          |
| 因素分析、归因     | 线性模型 / 凹凸性            |
| 权力结构、利益分配 | 价值与权力模型               |
| 关系网络、信息流   | 网络模型                     |
| 传播、扩散、流行   | 广播/扩散/传染病             |
| 不确定性、信息量   | 熵                           |
| 时序、趋势判断     | 随机游走 / 马尔可夫模型      |
| 历史锁定、路径分析 | 路径依赖                     |
| 局部→整体涌现      | 局部互动模型                 |
| 系统稳定性         | 李雅普诺夫函数 / 系统动力学  |
| 集体行为、临界点   | 阈值模型                     |
| 竞争、合作、策略   | 博弈论 / 合作模型 / 机制设计 |
| 搭便车、公地悲剧   | 集体行动                     |
| 信息不对称         | 信号传递模型                 |
| 学习、适应         | 学习模型 / 多臂老虎机        |
| 复杂优化、创新     | 崎岖景观模型                 |
| 选址、定位、差异化 | 空间与享乐选择               |

### Step 3: 解释选择

向用户说明为什么选这个模型，包括：

- 该模型的核心思想（一句话）
- 为什么适合当前问题
- 该模型的局限性（什么情况下不适用）

### Step 4: 应用模型分析

按照该模型的框架逐步分析。输出结构需符合模型特点（见 `references/models.md` 中各模型的"输出结构"）。

**通用分析框架**（根据具体模型调整）:

```
## 模型分析：{模型名称}

### 问题定义
{简述要分析的问题}

### 模型框架应用
{按照模型的步骤/维度逐一分析}

### 关键洞察
{模型揭示的核心发现}

### 局限性
{该模型不覆盖的方面}
```

### Step 5: 综合分析与建议

模型分析之后，跳出模型框架进行更全面的分析：

**覆盖多维度:**

- 实用性建议：基于模型分析得出的可操作行动方案
- 风险与权衡：模型揭示的风险和其他未被模型覆盖的风险
- 替代方案：如果模型分析指向不明确，给出替代路径

不要因为用了模型框架就让分析变窄。**模型是分析透镜，输出仍需全面。**

### Step 6: 补充多模型视角（可选）

如果问题复杂，在主要分析之后简要补充另一个模型的视角：

```
### 补充视角：{第二个模型}
{该模型补充了哪些不同角度的洞察}
```

## 输出格式

始终输出结构清晰的 Markdown，包含：

1. 推荐模型及选择理由
2. 完整的模型分析
3. 关键洞察和行动建议

如果用户后续有更多问题，可以继续深入分析或换用其他模型。

## 重要提醒

- **不要创造书中没有的模型**。只使用 `references/models.md` 中列出的 24 个模型。
- **解释清晰**。每个模型的概念要用通俗语言解释，不要假设用户熟悉这些模型。
- **诚实面对局限**。明确告诉用户模型分析是简化，真实情况更复杂。
- **中文为主**。模型名称可中英对照，但分析内容用中文。
